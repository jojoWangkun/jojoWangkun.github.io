GitHub Pages部署步骤

创建GitHub账户

如果您还没有GitHub账户，请先注册一个

创建新仓库

登录GitHub，点击右上角"+"，选择"New repository"

仓库名称格式: username.github.io (username替换为您的用户名)

上传文件

将下载的HTML文件重命名为index.html并上传到仓库

或者直接拖拽文件到仓库页面

启用GitHub Pages

进入仓库的"Settings" → "Pages"

在"Source"中选择"main"分支，点击"Save"

访问您的网站

等待几分钟后，访问: https://username.github.io

(将username替换为您的GitHub用户名)

注意: 仓库必须设置为公开(Public)，否则GitHub Pages无法工作。
创建GitHub仓库

故障排除 - 解决404错误

如果您遇到404错误，请检查以下事项:

  • 确认仓库名称格式为username.github.io（username是您的GitHub用户名）
  • 确认仓库 visibility 设置为 Public（而不是Private）
  • 确认HTML文件已命名为index.html并位于根目录
  • 在仓库Settings中已启用GitHub Pages并选择main分支作为源
  • 等待5-10分钟让GitHub完成部署
